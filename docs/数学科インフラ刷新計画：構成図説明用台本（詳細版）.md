# **数学科インフラ刷新計画：新構成図（v1.7）詳細説明台本**

## **1\. 導入（プロジェクトの全体像）**

「現在、数学科では16台の老朽化した物理サーバーを運用していますが、これらを2台の最新物理サーバー（ThinkSystem SR250 V3）へ集約・統合します。今回の刷新の柱は『1人運用の省力化』と『多要素認証（MFA）による鉄壁のセキュリティ』の両立です。」

**【専門用語解説：サーバーの集約】**

本来16台必要だった仕事を、1台の高性能なサーバーの中で「仮想的な小部屋（コンテナ）」に分けて動かすことで、電気代や故障リスクを減らすことです。

## **2\. ユーザーとアクセス方法（図の上段）**

「図の左上、\*\*【エンドユーザー＆デバイス】\*\*をご覧ください。

これまで課題だったeFERECによるブラウザ認証を廃止し、**802.1X認証**へと移行します。

* **研究室の有線LAN**は、コンセントに挿すだけでOSが自動的に認証を行います。  
* \*\*Wi-Fi（CHUO-MATH）\*\*も、WPA Enterprise方式により、一度設定すればスマホやPCを開くだけで接続されます。  
* ブラウザを持たない**プリンター**などは、MAC認証（MAB）という仕組みを使い、管理者が事前に登録した機器だけを通すようにします。」

**【専門用語解説：802.1X（ハチマルニテンイチエックス）認証】**

ブラウザでログイン画面を出さずに、パソコンのOS自体が「私は数学科のメンバーです」と証明書やIDを提示してネットワークに繋いでもらう、世界標準の安全な接続方法です。

**【専門用語解説：MAC（マック）認証 / MAB】**

機器が持っている固有のシリアル番号（MACアドレス）を「合言葉」にして、プリンターなどの画面がない機器を識別し、接続を許可する仕組みです。

## **3\. ネットワークの境界と防衛（図の中段）**

「次に、図の中央にある\*\*【ネットワーク境界】\*\*です。

外部との通信は、次世代ファイアウォールの **Cisco Firepower 1140 (ASA)** がガードします。学外からのアクセスはVPNに限定します。

その下の **コアL3スイッチ** は、学内のVLAN管理と認証ゲートウェイを担います。」

**【専門用語解説：次世代ファイアウォール（ASA）】**

単なる「通す・通さない」だけでなく、通信の中身を見てウイルスや不正な動きを検知し、自動で遮断する高度な防火壁です。

**【専門用語解説：VPN（ブイピーエヌ）】**

自宅やカフェから、暗号化された「専用のトンネル」を通って、あたかも大学の研究室にいるかのように安全にアクセスする技術です。

**【専門用語解説：VLAN（ブイラン）】**

1つのスイッチを論理的に分割し、「学生用」「教職員用」「サーバー用」のように、互いに覗き見できない独立したネットワークに分ける技術です。

## **4\. インフラの心臓部（図の下段左：Windows基盤）**

「右下の\*\*【学内統合サーバー】\*\*の左側、**Windows Server 2025 基盤** です。

ここには、数学科の全ユーザー情報を管理する **Active Directory (AD)** を配置し、さらにネットワークの基盤となる **DNS、DHCP、RADIUS** を統合します。」

**【専門用語解説：Active Directory (AD)】**

ネットワーク上のユーザー名とパスワードを一括管理する「巨大な電話帳」のようなものです。

**【専門用語解説：DNS / DHCP】**

DNSは「ドメイン名（名前）をIPアドレス（住所）に変換する案内板」、DHCPは「PCに自動でIPアドレス（住所）を割り振る役所」のような機能です。

**【専門用語解説：RADIUS（ラディウス）】**

Wi-Fiや有線LANの接続時に、AD（電話帳）を見て「この人は繋いでいい人か」を最終判定する審判のような役割です。

## **5\. アプリケーション層（図の下段右：Docker基盤）**

「右側の **Linux 統合基盤（Docker Compose）** です。

ここではメール、ファイル共有、プロキシ、そして自動化スクリプトが動きます。

* **Nginx Proxy Manager:** 全てのWebリクエストの入り口となります。**特筆すべきは、これまで毎年手作業で行っていたUPKI証明書の更新作業が不要になることです。** Let's Encryptという仕組みを使い、SSL証明書の取得と更新を全自動で行います。  
* **Authentik:** 今回の目玉です。AD認証に加えて\*\*多要素認証（MFA）\*\*を強制します。  
* **ad-to-mailcow (Pythonスクリプト):** 私が開発する自動化ロジックです。ADの変更を監視し、メールアカウントの作成・削除を自動化します。」

**【専門用語解説：Docker（ドッカー） / コンテナ】**

アプリを実行するために必要なものをすべて「コンテナ（箱）」に詰め込み、OSを汚さずにどこでも同じように動かせるようにする技術です。

**【専門用語解説：SSL証明書の自動更新（UPKIからの脱却）】**

サイトのURLを「https://」で安全に通信するための電子証明書です。従来は毎年書類を書いて設定していましたが、これを無料かつ自動で行う仕組み（Let's Encrypt）を導入し、管理の手間をゼロにします。

**【専門用語解説：多要素認証 (MFA)】**

パスワードだけでなく、スマホに届く通知や指紋などを組み合わせて、本人であることを2重3重に確認する、現在最も推奨されるセキュリティ対策です。

## **6\. 結び（この構成のメリット）**

「この構成により、ユーザーは『1つのID/PW』で全てのサービスを利用でき、管理者は『1枚の設計図（コード）』で全体を制御できます。面倒だった証明書更新などの定常業務も自動化されます。

将来的にメールだけを学外（さくらVPS等）へ移動させることも容易な、拡張性の高いアーキテクチャです。以上で説明を終わります。」

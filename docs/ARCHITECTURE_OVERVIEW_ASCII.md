# 数学科インフラ 構成図 (ASCII版 v1.0)

この図は、学内（オンプレミス）と学外（さくらVPS）の役割分担、およびデータの流れをテキスト形式で示しています。

## 全体構成図

```text
       ┌────────────────────────────────────────────────────────┐
       │   オンプレミス・サーバー (Single Host: Docker 基盤)    │
       │                                                        │
       │   ┌──────────────┐         ┌──────────────────────┐    │
       │   │  リバースプロキシ │ HTTPS   │    Nextcloud (ストレージ) │    │
       │   │   (NPM)      ├─────────▶│    ＆ Mailcow (メール)    │    │
       │   └──────┬───────┘         └──────────────────────┘    │
       │          │                            ▲                │
       │   ┌──────▼───────┐         ┌──────────┴───────────┐    │
       │   │    Authentik  │  LDAP   │ Windows Server 2025  │    │
       │   │  (MFA/Portal) ├─────────▶│     (AD / IDマスター)   │    │
       │   └──────────────┘         └──────────┬───────────┘    │
       │                                       │                │
       │   ┌───────────────────────────────────┘                │
       │   │         (内部ネットワーク同期)                     │
       │   ▼                                                    │
       │   ┌──────────────┐         ┌──────────────────────┐    │
       │   │ ad-to-mailcow│   API   │   既存継承コンテナ       │    │
       │   │ (同期スクリプト)├─────────▶│ (DNS/DHCP/RADIUS/EF) │    │
       │   └──────────────┘         └──────────────────────┘    │
       └────────────────────────────────────────────────────────┘
                    │                          │
                    ▼                          ▼
               [学内外利用者]                [学科内LAN]
```

## 役割解説

1. **オンプレ側（上段）**: IDマスター（AD）と、重いデータの処理（Nextcloud）を担当。
2. **VPS側（下段）**: メール（Mailcow）を担当し、学内が停電しても動き続ける。
3. **接着剤**: Pythonスクリプト (`ad-to-mailcow`) が、学内のADから学外のメールへIDを流し込みます。

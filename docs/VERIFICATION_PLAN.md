# インフラ検証環境の変更計画

## 概要
当初、リモートVPS（さくらVPS）を使用する予定だった検証フェーズにおいて、VPSをローカルの **Mac PC** に置き換えて実施します。これにより、インターネット経由の接続をシミュレートしつつ、物理的に管理可能な範囲でインフラの動作確認を行います。

## 検証環境の構成（変更後）
1.  **オンプレミス側（シミュレーション）**:
    *   新メインサーバー (Lenovo SR250 V3 想定)
    *   Dockerコンテナ群 (Mailcow, Authentik, etc.)
2.  **VPS側（Mac PCによるシミュレート）**:
    *   **Mac PC** を外部ノードとして使用。
    *   役割: リモートバックアップ先、外部メールリレー、またはセカンダリDNS等のシミュレーション。

## 準備タスク
- [ ] **Mac PC の環境構築**
    - [ ] Docker Desktop または OrbStack のインストール。
    - [ ] `docker-compose` 等のツール群の整備。
- [ ] **ネットワーク構成の定義**
    - [ ] ローカルネットワーク内での固定IP割り当て。
    - [ ] Mac PC 上での検証用サービス（SSHサーバー、バックアップストレージ等）の起動。
- [ ] **接続検証**
    - [ ] ローカル環境のメインサーバーから Mac PC への接続確認。
    - [ ] VPSとしての役割を果たすコンテナのデプロイ。

## 今後のスケジュール
1.  ローカルの検証用Macのセットアップ。
2.  ADからMailcowへの同期アプリ (`sync-apps/ad-to-mailcow`) の検証をこのMac上で行えるように調整。
3.  ハイブリッド構成全体（オンプレサーバー + Mac）の統合テスト。
